User Based Collaborative Filtering¶

As part of a project in my Big Data Analytics course, I was challenged to create a custom analysis of data provided by the instructor. The dataset is a subset of Amazon item ratings and can be found here. Working with Amazon data felt like the perfect opportunity to give a recommendation system a try, even if this method wasn't going to be covered in my course work. Ashay Pathak's post on Medium was a huge help in teaching me the ideas behing user-based and item-based collaborative filtering. He does a great job of walking through the reasoning in recommendation systems and showing snippets of his code. In an attempt to review recommendation systems for myself, I will be giving my interpretations of what he taught in his post within this project.

Normalize Ratings¶

In order to properly compare ratings across different users, some form of normalization is needed. Users might rate using different internal scales. For example, someone's rating of 3 might be equal to another rating of 2.

To normalize the ratings, I will take the difference of a user's rating and their overall average rating. To do this, I first calculate average ratings for each user. This average column is then joined onto the original user ratings data frame using the pandas merge function. It's important to remember that merge is used in pandas for what most people understand as a join in SQL.

Filter data to highly rated items and active users¶

After my first iteration, I realized users and items with too few ratings were impacting the calculation of similarity between any two users. Preprocessing for the similarity calculation involves replacing missing values with average user ratings, for users and items with few ratings, the average was dominating the calculation. For the sake of this project, I wanted to find cases of active users to find likely similarities. The process of filtering items with few ratings and unactive users is below.

First, I calculate a quick aggregation of count of ratings per item.

Cosine Similarity¶

The idea behind user-based collaborative filtering is to recommend items to users based on the items other similar users like. For example, if you and I rate the same 2 items very highly, I will probably like a 3rd item you like that I haven't yet rated. To find this similarity between users, we need one record per user with the score they gave to each item as the columns. To do this, a pivot table is created of userId, ItemId, and the ratings for each combination. These vectors will the be used to calculate similarity between users. Cosine similarity measures the cosine of the angle between two vectors.

Top k neighbors per user¶

With the similarity matrix created, we now need the capability to select the top n similar users. We do not want to recommend items to user that a distantly similar user has rated highly. The function find_n_neighbors() sorts a given data frame and keeps the first n columns.

In [383]:
def find_n_neighbors(df,n):
    order = np.argsort(df.values, axis=1)[:, :n]
    df = df.apply(lambda x: pd.Series(x.sort_values(ascending=False)
           .iloc[:n].index), axis=1).reset_index()
    return df

def scores(user,item):
    #the input user's similar users
    similar_users = similar_user_k50[similar_user_k50.userId==user].values
    
    #the input user average rating: ru
    user_avg_rating = means.loc[means.userId == user]['rating']
    
    #pull ratings and users from the main data for the particular item, rv's
    item_ratings = final_filter.loc[:,item]
    
    # Filter to items actually rated by users. Not the NaN replaced with average
    item_ratings_RealRating = item_ratings[item_ratings > -99]
    
    #Check if the users whose ratings were pulled are in your similar users
    other_users_similar = item_ratings_RealRating[item_ratings_RealRating
                                                  .index.isin(similar_users.squeeze())]
    
    if(len(other_users_similar)==0):
        return "Please rate more items"
    
    other_users_similarNull = other_users_similar[other_users_similar.notnull()]
    
    #take the value of the index, which is the userId
    other_users =  other_users_similarNull.index.values
    
    similarity_with_other_users = similarity.loc[user,other_users]
    
    similarity_ScaledRating = pd.concat([similarity_with_other_users,item_ratings], axis=1,sort=False).reset_index()
    similarity_ScaledRating.columns = ['userId','sim','scaledRating']
    similarity_ScaledRating['weighted'] = similarity_ScaledRating['sim'] * similarity_ScaledRating['scaledRating']
    sum_weighted_ratings = similarity_ScaledRating.weighted.sum()
    sum_weights = similarity_ScaledRating.sim.sum()
    

    result = user_avg_rating + (sum_weighted_ratings/sum_weights)

    return result

def recommend(user):
    from collections import Counter
    itemsRated = set(ratings_filtered.loc[ratings_filtered.userId == user]['itemId'])
    similarUsers = similar_user_k15[similar_user_k15.userId==user].values
    
    itemsSimilarUsersHaveRated = ratings_filtered.loc[ratings_filtered.userId.isin(similarUsers.squeeze())]['itemId']
    
    #counter to find what item other users have reviewed the most
    c = Counter(itemsSimilarUsersHaveRated)
    m = max(c, key=c.get)
    
    itemsSimilarUsersHaveRated = set(itemsSimilarUsersHaveRated)
    itemsYouNotRated = itemsSimilarUsersHaveRated - itemsRated
    itemsYouNotRated = list(itemsYouNotRated)
    my_scores = []
    for item in itemsYouNotRated:
        my_scores.extend(scores(user,item))
    data = pd.DataFrame({'scores':my_scores,'items':itemsYouNotRated})
    
    items = data.sort_values(by='scores',ascending=False).head(5)['items']
    
    return items, c, m
